## 커밋 전 검증 순서

```
1. npm run build        # TypeScript 컴파일 오류 없는지 확인
2. npm run test:e2e     # E2E 스모크 테스트 통과 확인
```

- 두 단계 모두 통과해야 커밋을 진행한다
- 실패 시 반드시 수정 후 재확인한다
- 빌드/테스트 오류를 무시하고 커밋하지 않는다

## 스모크 테스트 범위 (`e2e/smoke.spec.ts`)

- 홈 `/projects` 진입 시 프로젝트 카드 목록이 렌더됨
- 카드 클릭 시 상세 페이지(`/projects/:id`)로 이동하고 제목이 표시됨

이 두 시나리오는 앱 핵심 흐름(목록 조회 → 상세 이동)이 살아있음을 보증한다.
기능 추가 시 스모크 테스트를 수정하지 않아도 된다 — 핵심 흐름이 깨지지 않는 한.
